Label

label creates a simple on-screen text label. Users can interact with the label via mouse down events if a channel name has been assigned.

List of supported properties

Active | Automatable | Bounds | Channel | Colour | Font | Opacity | Text | Visible

Unique Properties​

Active​

"active": 0

Will deactivate a control if 0 is passed. Controls which are deactivated can still be updated from Csound.

Bounds​

"bounds": {"left":0, "top":0, "width":100, "height":100}

Integer values that set position and size on screen(in pixels).

Channel​

"channel": "chan"

The unique channel name associated with a widget/component. This unique name is used to get and set widget properties.

In Cabbage 3, channels don't need to start with a letter, and white spaces are permitted.

Colour​

"colour": {
    "fill": "#0295cf",
    "stroke": {
        "colour": "#dddddd",
        "width": 1
    }
}

Colour properties can be any valid CSS or HTML color string. They can also accepts be hexadecimal strings, or RGB / RGBA array, where each channel ranges from 0 to 255. For example, "colour":{"fill":[0, 0, 255]} or "colour":{"fill":""#ff0000"}.

The stroke object set the outline colour and width for any widget that supports it.

Font​

"font": {
    "family": "Verdana",
    "size": 0,
    "align": "centre",
    "colour":"#ff0000"
}

Sets the font size, family, colour and font alignment of the widget, where applicable.

Opacity​

"opacity": 1

The opacity attribute property can vary between 0 (fully transparent) and 1 (fully opaque).

Text​

"text": "string"

"string" will be the text that appears on the widget.

Visible​

"visible": 1

A value of 0 will cause the widget to become invisible. Widgets have their visibility set to 1 by default.

Automatable​

"automatable": 1

Defaults to 1. Determines if a widget is automatable by a host DAW. Automatable widgets show up as plug-in parameters in the host. Non-automatable widgets can still communicate with Csound but are not accessible by the host. Note that hosts don't allow this parameter to change dynamically. If you change this setting, the plugin will need to be reloaded.

Example​

<Cabbage>[
    {"type":"form","caption":"Label Example","size":{"width":580,"height":500},"pluginId":"def1"},
    {"type":"label","channel":"label2","bounds":{"left":158,"top":37,"width":228,"height":21}, "text":"Don't label me!!", "channelType":"number", "automatable":1}
]</Cabbage>
<CsoundSynthesizer>
<CsOptions>
-n -d -+rtmidi=NULL -M0 -m0d --midi-key=4 --midi-velocity-amp=5
</CsOptions>e
<CsInstruments>
; Initialize the global variables. 
ksmps = 32
nchnls = 2
0dbfs = 1


; Rory Walsh 2021 
;

; even for commercial purposes, all without asking permission. 

instr 1
 kLabel cabbageGetValue "label2"
 printk2 kLabel
endin


</CsInstruments>
<CsScore>
;causes Csound to run for about 7000 years...
i1 0 z
</CsScore>
</CsoundSynthesizer>
